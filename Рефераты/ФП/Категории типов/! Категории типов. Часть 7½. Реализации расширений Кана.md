
# Возможности расширений Кана

Сама концепция расширений Кана функтора $F$ вдоль $G$ опирается на идею *продолжения вычислений*. Функтор расширения Кана продолжает функтор $G$ так, чтобы их композиция дала «что-то похожее на $F$». При этом функториальность расширения сосредотачивается в функции-продолжении.

Давайте посмотрим на экземпляры класса типов `Functor[F[_]]` для расширений Кана:
```scala
given ranFunctor: [F[+_], G[+_]] => Functor[F / G] =  
  [A, B] => (f: A => B) => (fga: (F / G)[A]) =>  
    [R] => (cont: B => G[R]) =>  
      fga(cont `compose` f)  
      //  ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑  
  
given lanFunctor: [F[+_], G[+_]] => Functor[G \ F] =  
  [A, B] => (f: A => B) => (fga: (G \ F)[A]) =>  
    [R] => (cont: [X] => (G[X] => B) × F[X] => R) =>  
      fga([X] => (fxaGx: (G[X] => A) × F[X]) =>  
        cont(f `compose` fxaGx._1, fxaGx._2))  
        //   ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑
```

Более сложный код для $\mathrm{Lan}_GF$ — неизбежная плата за работу с экзистенциальными (алгебраическими) типами данных, и механизм сопоставления с шаблонами способен лишь усложнить её дополнительным синтаксисом.

Самое интересное то, что в обеих реализациях не задействована функториалность `F[_]` и `G[_]`. «Поднимаемая» функция `f: A => B` просто композируется либо с продолжением, либо внутри него. Такая особенность позволяет оптимизировать цепочки контейнерных вычислений для представлений Йонеды и монады коплотности, которые мы рассмотрим далее.

Единица и коединица наших расширений Кана реализуются очень просто, с использованием тождественной функции `identity`: 
```scala
def εRan[F[+_], G[+_]]: (F / G) ∘ G ~> F =  
  [A] => ran => ran(identity[G[A]])
  
def ηLan[F[+_], G[+_]]: F ~> ((G \ F) ∘ G) =  
  [A] => (fa: F[A]) =>
    [_] => cont => cont(identity[G[A]] -> fa)
```

Также крайне полезны универсальные свойства, сопоставляющие кандидатам $(H, \alpha)$ универсальные естественные преобразования:
```scala
def uRan[F[+_], G[+_], H[+_]: Functor](α: H ∘ G ~> F): H ~> (F / G) =  
  [A] => (ha: H[A]) =>  
    [R] => (cont: A => G[R]) =>  
      α(fmap[H](cont)(ha))  
  
def uLan[F[+_], G[+_], H[+_]: Functor](α: F ~> (H ∘ G)): (G \ F) ~> H =  
  [A] => (lana: (G \ F)[A]) =>  
    lana[H[A]]([X] => gxaFx => fmap[H](gxaFx._1)(α(gxaFx._2)))
```

# Йонеда

В шестой части обзора [анонсировались](https://habr.com/ru/articles/1012402/#yoneda) представления Йонеды функтора, как его расширений Кана вдоль тождественного $Id$. В частности:
```scala
type Yoneda[F[+_]] = F / Id // [A] =>> [X] => (A => X) => F[X]
```

Замечательной особенностью представления Йонеды является то, оно изоморфно исходному функтору. Действительно, ветви изоморфизма образуют единица расширения вместе с универсальным морфизмом кандидата $(H, \alpha)=(F, Id_F)$:


```scala
type Coyoneda[F[+_]] = Id \ F  // [A] =>> [R] => ([X] => (X => A) × F[X] => R) => R
```
